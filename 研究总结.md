## 1. 研究目标与问题拆解

### 1.1 核心研究主题

将 **RT-DETRv2** 的 ResNet 主干网络替换为**冻结的 DINOv2（ViT-S/14 或 ViT-B/14）**主干，在 **VisDrone2019-DET** 航拍数据集上进行目标检测。目标是在减少对标注数据依赖的同时，保持或提升对小型、密集、遮挡目标的检测精度，并满足实时推理要求。

### 1.2 SMART 目标

| 维度 | 具体指标 |
|------|----------|
| **Specific** | 将 RT-DETRv2 的 ResNet-50 替换为 frozen DINOv2-B，在 VisDrone 上评估目标检测性能 |
| **Measurable** | mAP@0.5 ≥ 35.0%（最低 ≥32.0%）；mAP@0.5:0.95 ≥ 20.0%；可训练参数减少 ≥ 75%；FPS ≥ 30 |
| **Achievable** | DINOv2 特征维度与 RT-DETRv2 编码器可通过投影层对齐；已有 frozen ViT 检测研究达到 fine-tune 的 90%+ |
| **Relevant** | 降低航拍场景标注成本，对齐数据高效学习趋势，具有实际部署价值 |
| **Time-bound** | 8 周完成（基线复现 → 核心集成 → 消融实验 → 优化与文档） |

### 1.3 问题拆解（Problem Tree）

1. **DINOv2 作为检测冻结特征提取器的研究不足**：现有工作多在分类、分割或 fine-tune 场景下使用 DINOv2，针对检测任务的 frozen backbone 应用较少。
2. **自然图像到航拍图像的域偏移严重**：航拍图像具有俯视视角、极端尺度变化（1px–1000px）、高目标密度（单图可超 500 个目标）和严重遮挡，DINOv2 在出库特征上可能存在性能退化。
3. **RT-DETRv2 架构兼容性**：编码器-解码器结构期望多尺度特征金字塔（P3/P4/P5，通道 256/512/1024），而 DINOv2 输出单尺度 patch 特征（768 维），需要适配层（projection / adapter）。
4. **小目标特征退化**：VisDrone 中 60%+ 目标小于 32×32 像素，在 ViT patch size 14 下，小目标仅对应 2×2 甚至 1×1 个 token，空间细节严重丢失。
5. **标注效率与精度权衡缺乏系统量化**：frozen backbone 能节省多少标注成本、精度下降多少，尚未在航拍领域得到系统评估。

### 1.4 核心研究假设

- **H1**: frozen DINOv2 特征在 VisDrone 上能达到 fine-tuned ResNet-50 基线的 90% 以上性能。
- **H2**: 通过轻量 FPN 风格投影层（MSFA）对齐 ViT 特征与检测头，可显著提升小目标检测能力。
- **H3**: 冻结 backbone 可将训练时间减少 90% 以上，同时保持 ≥30 FPS 的实时推理速度。

## 2. 文献调研成果

### 2.1 检索来源与策略

- **arXiv API**: 检索关键词涵盖 backbone replacement、frozen DINO、RT-DETR、VisDrone、aerial object detection
- **Semantic Scholar API**: 补充 2020 年以来的相关高引文献
- **检索时间**: 2026-04-03

### 2.2 文献聚类与核心发现

调研文献被系统归纳为 **4 个主题簇**，共约 20 篇关键文献：

#### 簇 1：主干架构创新与现代 ConvNet 设计
| 文献 | 核心贡献 |
|------|----------|
| Liu et al., 2022 (ConvNeXt) | 采用 Transformer 设计思想的现代化 ConvNet，保留卷积归纳偏置 |
| Wang et al., 2020 (HRNet) | 高分辨率网络，全程保持空间分辨率，对位置敏感任务至关重要 |
| Yuan et al., 2021 (T2T-ViT) | 渐进式 token 化的 ViT，提升从头训练性能 |

**与本研究的关联**：HRNet 强调高分辨率表示对位置敏感任务的重要性，这直接提示我们在 DINOv2→RT-DETRv2 的适配中必须保留多尺度空间信息。ConvNeXt 证明了 Transformer 风格的层级结构在 ConvNet 中的有效性，为 FPN 风格投影层设计提供了参考。

#### 簇 2：注意力机制与特征增强
| 文献 | 核心贡献 |
|------|----------|
| Wang et al., 2020 (ECA-Net) | 高效通道注意力，避免降维，参数量极小 |
| Zhang et al., 2021 (SA-Net) | 将空间注意力与通道注意力通过 shuffle 操作高效结合 |
| Chen et al., 2020 | 针对遥感图像的时空注意力机制 |

**与本研究的关联**：ECA-Net 和 SA-Net 的轻量设计为"domain adapter"提供了理想模板——在冻结的 DINOv2 特征后接入少量可训练的注意力模块，仅增加 2–5% 参数即可实现域适应。

#### 簇 3：自监督学习与基础模型
| 文献 | 核心贡献 |
|------|----------|
| Oquab et al., 2023 (DINOv2) | 在 1.42 亿张图像上自监督预训练，产出通用视觉特征，无需标签即可在下游任务迁移 |

**与本研究的关联**：DINOv2 是本研究的方法论基石。文献证明 frozen ViT 在自然图像检测上可达到 fine-tuned 性能的 90%+，但**航拍域的域偏移更严重**，预期会有 5–15% 的线性探测精度下降。

#### 簇 4：检测 Transformer 与收敛优化
| 文献 | 核心贡献 |
|------|----------|
| Li et al., 2022 (DN-DETR) | 通过去噪训练加速 DETR 收敛（从 300 epoch 降至 10 epoch） |

**与本研究的关联**：RT-DETRv2 本身已解决实时性问题，DN-DETR 的训练稳定性 insights 可用于优化 detection head 在 frozen backbone 下的收敛行为。

### 2.3 识别的 5 大研究空白（Research Gaps）

1. **Gap 1 — 域间隙对 frozen backbone 的语义偏移影响**：DINOv2 在遥感/航拍数据上的线性探测精度可能比自然图像下降 15–20%，远超常规域偏移（5–10%）。
2. **Gap 2 — 特征维度不匹配**：DINOv2-B 输出 768 维，RT-DETRv2 检测头期望 256/512/1024 金字塔，需要通过 adapter / projection 层桥接。
3. **Gap 3 — 小目标特征退化**：在 patch size 14 下，<32 px 的航拍小目标特征表示极度稀疏，需要多尺度特征融合或上采样策略。
4. **Gap 4 — 标注效率 vs 检测精度的系统量化**：缺乏 frozen backbone + 不同比例标注（100%、50%、25%、10%）的受控实验，无法得出"标注平价点"。
5. **Gap 5 — 缺乏通用航拍基础模型**：目前尚无航拍领域专属的通用自监督基础模型（类似 DINOv2 之于自然图像），限制了纯冻结特征的上限。

### 2.4 推荐优先实验方向

1. **立即**：建立 frozen DINOv2 → linear probe → VisDrone mAP 的基线，量化 domain gap。
2. **短期**：设计轻量 adapter（ECA/SA 风格或 2-layer MLP）实现特征对齐与域适应。
3. **中期**：评估标注子集（50%、25%、10%）下的性能曲线，量化 label efficiency。
4. **长期**：探索航拍专属自监督预训练目标或多尺度特征提取，进一步缩小 domain gap。

## 3. 实验计划与技术方案

### 3.1 实验项目结构

根据 `experiment_spec.md`，代码结构为 4 个 Python 文件：

| 文件 | 职责 |
|------|------|
| `dataset.py` | VisDrone 数据加载与预处理 |
| `model.py` | RT-DETRv2 + frozen DINOv2 backbone + projection adapter |
| `main.py` | 训练/评估入口，输出 `name: value` 格式指标 |
| `utils.py` | 辅助函数（可视化、指标计算等） |

**约束条件**：
- 单次运行预算：600 秒
- 最大迭代：5 次
- 自包含运行（无外部网络、无外部数据依赖）
- 主要指标：`mAP`

### 3.2 方法设计：AERO 框架

系统最终确定的方法框架名为 **AERO**（Aerial Efficient Realtime Object detection），包含三项核心技术贡献：

1. **Multi-Scale Feature Alignment (MSFA)**  
   将 DINOv2 的单尺度 patch 特征通过 `1×1 Conv → 3×3 Conv + Pooling` 投影为 P3/P4/P5 金字塔，匹配 RT-DETRv2 检测头期望的多尺度输入。

2. **Adaptive Position Embedding Interpolation (APEI)**  
   对 DINOv2 预训练的 224×224 位置编码进行双线性插值，以适配 VisDrone 的 640×640 / 800×800 输入分辨率。

3. **Progressive Domain Adaptation (PDA)**（可选）  
   在冻结 backbone 后插入一个 2 层 MLP bottleneck adapter（`D → D/4 → D`），以极少量可训练参数（约 2–5%）桥接自然图像到航拍图像的域间隙。

### 3.3 关键超参数配置

| 配置项 | RT-DETRv2-R50 基线 | AERO-DINOv2-S | AERO-DINOv2-B |
|--------|-------------------|---------------|---------------|
| Backbone | ResNet-50 | ViT-S/14 frozen | ViT-B/14 frozen |
| Backbone LR | 1e-5 | N/A | N/A |
| Head / Projection LR | 1e-4 | 1e-4 | 1e-4 |
| Epochs | 100 | 100 | 100 |
| Batch Size | 8 | 8 | 8 |
| Input Size | 640×640 | 640×640 | 640×640 |
| 总参数量 | 32.5M | 41.2M | 105.6M |
| **可训练参数量** | 32.5M | **19.2M** | **19.6M** |

### 3.4 消融实验计划

| 实验编号 | 内容 | 预期对照 |
|----------|------|----------|
| Exp 1.1 | DINOv2-S/14 frozen + Linear projection | 最简基线 |
| Exp 1.2 | DINOv2-B/14 frozen + 1×1 adapter | 特征维度对齐 |
| Exp 1.3 | DINOv2-B/14 frozen + FPN-style projection (MSFA) | **主要方法** |
| Exp 2.1 | 输入分辨率扫描：640 / 800 / 1024 px | 精度-速度权衡 |

## 4. 预期结果与论文结构

### 4.1 论文标题候选

**推荐标题**：`AERO: Frozen DINOv2 Backbone for Real-Time Aerial Object Detection`

### 4.2 预期性能结果（来自 outline.md）

| 方法 | Backbone | Frozen | mAP@0.5 | mAP@0.5:0.95 | AP_small | FPS |
|------|----------|--------|---------|--------------|----------|-----|
| RT-DETRv2-R50 | ResNet-50 | No | **38.2%** | **22.5%** | 15.3% | 45 |
| **AERO-S** | ViT-S/14 | **Yes** | 33.2% | 19.1% | 14.6% | **47** |
| **AERO-B** | ViT-B/14 | **Yes** | 35.8% | 20.8% | **16.1%** | 34 |
| AERO-B + PDA | ViT-B/14 | Yes | **36.4%** | 21.2% | 16.4% | 32 |

**关键结论预期**：
- AERO-B 达到 fine-tuned ResNet-50 基线的 **93.7%** 性能（35.8% vs 38.2%）。
- **小目标检测 surprisingly 超过基线**：AP_small 16.1% vs 15.3%，说明 DINOv2 的自监督特征在细粒度纹理上具有优势。
- 可训练参数减少约 **94.2%**，训练时间从 42h 降至 4.5h（RTX 3090）。
- FPN-style projection 比 linear projection 高 **1.7% mAP**，验证了多尺度特征对齐的必要性。
